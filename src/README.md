# توثيق مشروع طاولة اللحام الآلية
# Automated Welding Turntable Documentation

**اللوحة:** Arduino Mega 2560
**الواجهة:** Node-RED Dashboard / Vue `ui_template`
**الملفات الأساسية:** `src/main.cpp` و `src/node_red_dashboard.vue`

---

## 1. فكرة النظام

هذا المشروع يتحكم بطاولة لحام دوّارة تعمل بمحرك خطوي، وتتبادل الإشارات مع روبوت لحام تعاوني. المشغل يحدد مواضع اللحام من الواجهة، ثم يبدأ البرنامج. بعد ذلك تنتظر الطاولة إشارة الروبوت، تتحرك إلى كل موضع بالتسلسل، ترسل نبضة للروبوت ليبدأ اللحام، ثم تنتظر إشارة انتهاء اللحام قبل الانتقال للموضع التالي.

النظام يدعم الآن التوقف المؤقت، الاستمرار من نفس المكان، الذهاب إلى موضع محفوظ، الإيقاف مع حفظ الهوم، الترجمة، وعدة حمايات للواجهة حتى لا يعمل الزر في الوقت الخاطئ.

## 3. التوصيلات الرئيسية

| المنفذ | الاتجاه | الجهاز | الوظيفة |
|---:|---|---|---|
| `5` | Output | TB6600 `PUL+` | نبضات المحرك الخطوي. |
| `6` | Output | TB6600 `DIR+` | اتجاه دوران المحرك. |
| `7` | Output | TB6600 `ENA+` | تفعيل المحرك، حيث `LOW` تشغيل. |
| `8` | Input | Homing Sensor | حساس نقطة الصفر. |
| `9` | Output | Relay to Cobot | إرسال نبضة للروبوت. الريلاي Active-Low: مفصول افتراضياً `HIGH` ويعمل فقط عند الإرسال `LOW`. |
| `10` | Input | Relay from Cobot | استقبال إشارة الروبوت، والإشارة الفعالة هي `LOW`. |
| `11` | Output | NeoPixel WS2812 | لمبة حالة النظام. |

> مهم: ريلاي إرسال الإشارة للروبوت لا يبقى موصولاً في الوضع الافتراضي. يبقى `OFF/HIGH`، ولا يصبح `ON/LOW` إلا أثناء نبضة الإرسال فقط.

## 5. أوامر Node-RED إلى Arduino

| الأمر | الوظيفة |
|---|---|
| `HOMING` | بدء عملية الهوم باستخدام الحساس. |

| `GO_HOME` | الرجوع إلى موضع الهوم الحالي. يعمل أيضاً أثناء `PAUSE`. |
| `GO_TO_POS <steps>` | الذهاب إلى موضع محفوظ بعدد الخطوات المحدد. |
| `MOVE_FORWARD` | حركة يدوية باتجاه `CW`. |
| `MOVE_BACKWARD` | حركة يدوية باتجاه `CCW`. |
| `HOLD` | إيقاف الحركة اليدوية أو إيقاف البرنامج بطريقة تتطلب Homing جديد في حالة البرنامج. |
| `ADD` | حفظ الموضع الحالي داخل مصفوفة Arduino. |
| `PAUSE_PROGRAM` | تبديل حالة البرنامج بين `PAUSE` و `CONTINUE`. |
| `STOP` | إيقاف البرنامج تماماً مع الحفاظ على الهوم. لا يعود البرنامج للعمل إلا بعد `START`. |
| `ESTOP` | إيقاف طوارئ، يفصل الريلاي والمحرك ويطلب Homing جديد. |
| `DISARM` | إلغاء انتظار الروبوت في حالة `ARMED`. |
| `START_PROGRAM <steps...>` | بدء البرنامج باستخدام المواضع الفعالة المرسلة من الواجهة. |
| `SET_SPEED <10-100>` | ضبط نسبة سرعة المحرك. |
| `SET_COBOT_TIMEOUT <seconds>` | ضبط مهلة انتظار الروبوت أو `0` لإلغاء المهلة. |
| `STATUS` | طلب حالة مختصرة من Arduino. |

أهم الرسائل المستخدمة في الواجهة:

| الرسالة | المعنى |
|---|---|
| `SYSTEM_READY` | Arduino بدأ التشغيل. |
| `STATUS|HOMED:x|PROG:x|PAUSED:x|POS:x|STATE:x` | حالة مختصرة دورية للواجهة. |
| `STEPS:<value>` | تحديث موضع المحرك الحالي بالخطوات. |
| `HOMED` / `STATUS:HOME_POSITION_0_STEPS` | تم تحديد الهوم بنجاح. |
| `STATUS:PROGRAM_ARMED` | البرنامج مسلح وينتظر إشارة الروبوت. |
| `STATUS:ROBOT_START_SIGNAL_RECEIVED` | وصلت إشارة بدء/استكمال من الروبوت. |
| `STATUS:PROGRAM_PAUSED` | البرنامج توقف مؤقتاً. |
| `STATUS:PROGRAM_RESUMED` | البرنامج استمر من مكان التوقف. |
| `STATUS:COBOT_FINISHED_SIGNAL_STORED_DURING_PAUSE` | وصلت إشارة الروبوت أثناء pause وتم حفظها. |
| `STATUS:RELAY_ON_TO_COBOT` | تم تفعيل ريلاي الإرسال للروبوت. |
| `STATUS:RELAY_OFF_TO_COBOT` | تم فصل ريلاي الإرسال بعد النبضة. |
| `STATUS:WAITING_FOR_COBOT_RESPONSE` | انتظار إشارة انتهاء اللحام من الروبوت. |
| `STATUS:CYCLE_COMPLETE_PIECE_DONE` | انتهت دورة قطعة كاملة. |
| `STATUS:PROGRAM_STOPPED_KEEP_HOME` | تم تنفيذ `STOP` مع الحفاظ على الهوم. |
| `ERROR:*` | خطأ أو حالة ممنوعة. |

## 7. واجهة التشغيل

### 7.1 جدول المواضع

- يعرض المواضع باسم `POS 1`, `POS 2`, ... حسب ترتيب الجدول الحالي.
- يمكن تفعيل/تعطيل كل موضع من زر القفل.
- يمكن تعديل عدد الخطوات، حذف الموضع، أو تغيير الترتيب للأعلى والأسفل.
- `SAVE` يحفظ قائمة المواضع كملف JSON.
- `LOAD` يسترجع المواضع من ملف JSON.
- المواضع محفوظة أيضاً في `localStorage` باسم `welding_positions`.

### 7.2 الأزرار الأساسية

| الزر | الوظيفة الحالية |
|---|---|
| `HOMING` | يعمل عندما لا يكون البرنامج شغالاً. يعيد تصفير النظام بالحساس ويصفر عداد القطع. |
| `PAUSE / CONTINUE` | يعمل بعد `START`. عند الضغط يصبح البرنامج متوقفاً في مكانه، ويتغير النص إلى `CONTINUE`. عند الضغط مرة أخرى يكمل من نفس النقطة بدون Homing. |
| `CW` / `CCW` | حركة يدوية. مسموحة قبل التشغيل أو أثناء `PAUSE`. لا تمسح ذاكرة إشارة الروبوت المحفوظة أثناء pause. |
| `HOLD` | يوقف الحركة اليدوية ويجعل الموضع جاهزاً للحفظ عبر `ADD`. |
| `ADD` | يحفظ الموضع الحالي بعد التوقف اليدوي. |
| `GO HOME` | يرجع الطاولة إلى الهوم. يعمل قبل التشغيل، ويعمل أيضاً أثناء `PAUSE`. |
| `SET HOME` | يجعل المكان الحالي هو الهوم. لا يعمل أثناء البرنامج. تظهر نافذة تأكيد قبل التنفيذ. |
| `GO TO POS` | يفتح نافذة اختيار موضع محفوظ والذهاب إليه. يعمل فقط عندما يكون النظام متوقفاً أو في `PAUSE`، ولا يعمل أثناء التشغيل الفعلي أو أثناء `ARMED`. |
| `STOP` | يوقف البرنامج تماماً مع الحفاظ على الهوم. بعده يجب ضغط `START` من جديد. لا يعمل أثناء `PAUSE`. |
| `START` | يفتح نافذة تأكيد بدلاً من `alert/confirm` التقليدي، ثم يرسل المواضع الفعالة فقط إلى Arduino. |
| `E-STOP` | إيقاف طوارئ، يفصل الإشارات ويجعل النظام يحتاج Homing جديد. |

### 7.3 نافذة GO TO POS

- تظهر قائمة منسدلة تحتوي أول خيار `---------` لإجبار المشغل على اختيار موضع.
- زر `GO` لا يعمل إلا بعد اختيار موضع حقيقي.
- أسماء المواضع في القائمة تعرض بالتسلسل حسب الجدول الحالي: `POS 1`, `POS 2`, ...
- زر `CLOSE` يغلق النافذة بدون حركة.

### 7.4 حالة الأزرار المعطلة

الأزرار التي لا يصح استخدامها في مرحلة معينة تظهر بلون باهت، ويتغير مؤشر الماوس إلى شكل غير مسموح (`not-allowed`) حتى يعرف المشغل أن الزر غير مناسب في تلك الحالة.

## 8. دورة العمل الحالية

1. شغل النظام وافتح واجهة Node-RED.
2. اضغط `HOMING` أو استخدم `SET HOME` لتحديد الهوم.
3. حرك الطاولة يدوياً باستخدام `CW/CCW`.
4. اضغط `HOLD` عند الموضع المطلوب.
5. اضغط `ADD` لحفظ الموضع.
6. كرر الخطوات لكل مواضع اللحام.
7. فعّل أو عطّل المواضع المطلوبة من الجدول.
8. اضغط `START`، ثم `OK` في نافذة التأكيد.
9. النظام يدخل `ARMED` وينتظر إشارة الروبوت.
10. عند وصول إشارة الروبوت، ينتظر النظام ثانيتين ثم يتحرك إلى أول موضع.
11. عند الوصول، يرسل Arduino نبضة ريلاي لمدة ثانية واحدة إلى الروبوت.
12. ينتظر النظام إشارة انتهاء اللحام من الروبوت.
13. بعد الإشارة ينتظر 3 ثوانٍ ثم ينتقل للموضع التالي.
14. بعد آخر موضع يرجع إلى الهوم، يزيد عداد القطع الملحومة، ثم يدخل `ARMED` مرة أخرى لانتظار القطعة التالية.

## 9. منطق PAUSE المهم

- `PAUSE` لا يعمل إلا بعد بدء البرنامج.
- عند الضغط على `PAUSE` يتوقف البرنامج في مكانه ويتحول الزر إلى `CONTINUE`.
- أثناء `PAUSE` يمكن استخدام `CW`, `CCW`, `HOLD`, و `GO HOME`.
- إذا وصلت إشارة انتهاء اللحام من الروبوت أثناء `PAUSE`، يتم حفظها في الذاكرة.
- عند الضغط على `CONTINUE`، إذا كانت إشارة الروبوت محفوظة، يكمل النظام مباشرة من النقطة الصحيحة ولا يحتاج إشارة جديدة.
- الحركة اليدوية أثناء `PAUSE` لا تمسح إشارة الروبوت المحفوظة.

---

## 10. منطق STOP و E-STOP

### STOP

- يعمل فقط أثناء عمل البرنامج وليس أثناء `PAUSE`.
- يوقف البرنامج كلياً ويفصل ريلاي الروبوت.
- يحافظ على حالة الهوم `isHomed` ولا يطلب Homing جديد.
- بعد `STOP` لا يعمل النظام تلقائياً، ويجب ضغط `START` من جديد.

### E-STOP

- إيقاف طوارئ كامل.
- يفصل ريلاي الروبوت ويوقف المحرك.
- يلغي التشغيل والانتظار و pause.
- يجعل النظام في حالة خطأ ويطلب Homing جديد.

## 11. مهلة انتظار الروبوت

الواجهة تحتوي إعداد مهلة انتظار الروبوت:

- `5 min`
- `15 min` وهو الافتراضي الحالي
- `30 min`
- `60 min`
- `none` لإلغاء المهلة

يتم إرسال الإعداد إلى Arduino بالأمر `SET_COBOT_TIMEOUT`. أثناء انتظار الروبوت، تعرض الواجهة العد التنازلي حتى الخطأ، أو تعرض أن المهلة غير مفعلة عند اختيار `none`.

## 14. ملاحظات تقنية

- كود Arduino يعتمد على أسلوب non-blocking باستخدام `millis()` و `AccelStepper.run()` حتى تبقى أوامر الطوارئ والتحديثات فعالة أثناء الحركة.
- ريلاي الروبوت Active-Low: التفعيل يتم بـ `LOW` والفصل بـ `HIGH`.
- بعد انتهاء كل دورة كاملة، يرجع النظام إلى الهوم ثم يعود إلى `ARMED` تلقائياً لانتظار القطعة التالية.
- عند تعديل المواضع أو حذفها أو تحميل ملف أثناء `ARMED`، ترسل الواجهة `DISARM` حتى لا يبدأ النظام من إشارة روبوت قديمة.
- `GO TO POS` يعتمد على خطوات الموضع المحفوظة في الواجهة ويرسلها مباشرة إلى Arduino.
- أسماء مواضع `GO TO POS` لا تعتمد على أسماء داخلية قديمة، بل على ترتيب الجدول الحالي.
