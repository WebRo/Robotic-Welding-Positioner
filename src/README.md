# توثيق مشروع طاولة اللحام الآلية
# Automated Welding Turntable Documentation

**اللوحة:** Arduino Mega 2560
**الواجهة:** Node-RED Dashboard / Vue `ui_template`
**الملفات الأساسية:** `src/main.cpp` و `src/node_red_dashboard.vue`

---

## 1. فكرة النظام

هذا المشروع يتحكم بطاولة لحام دوّارة تعمل بمحرك خطوي وتتبادل الإشارات مع روبوت لحام. يقوم المشغل بتحديد مواضع اللحام من الواجهة، ثم يضغط `START`. بعدها ينتظر النظام إشارة الروبوت، يتحرك إلى كل موضع بالتسلسل، يرسل نبضة للروبوت، ثم ينتظر إشارة انتهاء اللحام قبل الانتقال للموضع التالي.

يدعم النظام حالياً: `PAUSE/CONTINUE`، الذهاب إلى موضع محفوظ، `STOP` مع الحفاظ على الهوم، `E-STOP`، الترجمة، وتعطيل الأزرار غير المناسبة حسب حالة التشغيل.

## 3. التوصيلات الرئيسية

| المنفذ | الاتجاه | الجهاز | الوظيفة |
|---:|---|---|---|
| `5` | Output | TB6600 `PUL+` | نبضات المحرك الخطوي. |
| `6` | Output | TB6600 `DIR+` | اتجاه دوران المحرك. |
| `7` | Output | TB6600 `ENA+` | تفعيل المحرك، حيث `LOW` تشغيل. |
| `8` | Input | Homing Sensor | حساس نقطة الصفر. |
| `9` | Output | Relay to Cobot | إرسال نبضة للروبوت. الريلاي Active-Low: مفصول افتراضياً `HIGH` ويعمل فقط عند الإرسال `LOW`. |
| `10` | Input | Relay from Cobot | استقبال إشارة الروبوت، والإشارة الفعالة هي `LOW`. |
| `11` | Output | NeoPixel WS2812 | لمبة حالة النظام. |

## 4. إعداد السيريال

يجب أن يكون إعداد Node-RED مطابقاً لكود Arduino:

| الإعداد | القيمة |
|---|---|
| Baud rate | `115200` |
| Data bits | `8` |
| Stop bits | `1` |
| Parity | `none` |
| نهاية الأمر | `\n` newline |

تم رفع السرعة من `9600` إلى `115200` لتقليل تأثير السيريال على نعومة نبضات المحرك، خصوصاً على Linux/Radxa.

## 5. أوامر Node-RED إلى Arduino

| الأمر | الوظيفة |
|---|---|
| `HOMING` | بدء عملية الهوم باستخدام الحساس. |
| `SET_HOME_HERE` | جعل المكان الحالي هو الهوم بدون حساس. |
| `GO_HOME` | الرجوع إلى الهوم، ويعمل أيضاً أثناء `PAUSE`. |
| `GO_TO_POS <steps>` | الذهاب إلى موضع محفوظ بعدد الخطوات المحدد. |
| `MOVE_FORWARD` | حركة يدوية باتجاه `CW`. |
| `MOVE_BACKWARD` | حركة يدوية باتجاه `CCW`. |
| `HOLD` | إيقاف الحركة اليدوية وجعل الموضع جاهزاً للحفظ. |
| `ADD` | حفظ الموضع الحالي داخل مصفوفة Arduino. |
| `PAUSE_PROGRAM` | تبديل حالة البرنامج بين `PAUSE` و `CONTINUE`. |
| `STOP` | إيقاف البرنامج تماماً مع الحفاظ على الهوم. |
| `ESTOP` | إيقاف طوارئ، يفصل الريلاي والمحرك ويطلب Homing جديد. |
| `DISARM` | إلغاء حالة انتظار الروبوت في `ARMED`. |
| `START_PROGRAM <steps...>` | بدء البرنامج باستخدام المواضع الفعالة. |
| `SET_SPEED <10-100>` | ضبط نسبة سرعة المحرك. |
| `SET_ACCELERATION <5-100>` | ضبط نسبة تعجيل المحرك، والقيمة الافتراضية `30`. |
| `SET_COBOT_TIMEOUT <seconds>` | ضبط مهلة انتظار الروبوت أو `0` لإلغائها. |
| `STATUS` | طلب حالة مختصرة من Arduino. |

## 6. واجهة التشغيل

| الزر | الوظيفة |
|---|---|
| `HOMING` | يعمل عندما لا يكون البرنامج شغالاً، ويصفر النظام بالحساس. |
| `PAUSE / CONTINUE` | يوقف البرنامج في مكانه ثم يكمله من نفس النقطة بدون Homing. |
| `CW / CCW` | حركة يدوية قبل التشغيل أو أثناء `PAUSE`. |
| `HOLD` | يوقف الحركة اليدوية. |
| `ADD` | يحفظ الموضع الحالي بعد التوقف. |
| `GO HOME` | يرجع الطاولة إلى الهوم، ويعمل أثناء `PAUSE`. |
| `SET HOME` | يجعل المكان الحالي هو الهوم بعد نافذة تأكيد. |
| `GO TO POS` | يفتح نافذة اختيار موضع محفوظ ثم يذهب إليه. |
| `STOP` | يوقف البرنامج أو حركة `GO HOME / GO TO POS` مع بقاء الهوم محفوظاً. |
| `START` | يفتح نافذة تأكيد ثم يرسل المواضع الفعالة إلى Arduino. |
| `E-STOP` | إيقاف طوارئ كامل ويتطلب Homing جديد. |

الأزرار غير المسموحة في حالة معينة تظهر بلون باهت ويتغير مؤشر الماوس إلى `not-allowed`.
قيم `MOTOR SPEED` و `ACCELERATION` تحفظ في المتصفح وتدخل أيضاً في ملف الحفظ والتحميل.

## 7. منطق PAUSE

- يعمل `PAUSE` فقط بعد بدء البرنامج.
- عند الضغط عليه يتوقف البرنامج في مكانه ويتحول الزر إلى `CONTINUE`.
- أثناء `PAUSE` يمكن استخدام `CW`, `CCW`, `HOLD`, و `GO HOME`.
- إذا وصلت إشارة انتهاء اللحام من الروبوت أثناء `PAUSE` يتم حفظها.
- الحركة اليدوية أثناء `PAUSE` لا تمسح إشارة الروبوت المحفوظة.
- عند الضغط على `CONTINUE` يكمل النظام من المكان الصحيح بدون طلب إشارة جديدة إذا كانت الإشارة محفوظة.

---

## 8. STOP و E-STOP

### STOP

- يعمل أثناء تشغيل البرنامج، ويعمل أيضاً أثناء حركة `GO HOME` أو `GO TO POS`.
- يوقف البرنامج أو الحركة الحالية ويفصل ريلاي الروبوت.
- يحافظ على حالة الهوم `isHomed`.
- بعده يجب ضغط `START` من جديد.

### E-STOP

- إيقاف طوارئ كامل.
- يفصل ريلاي الروبوت ويوقف المحرك.
- يلغي التشغيل والانتظار و `PAUSE`.
- يجعل النظام يحتاج Homing جديد.

## 10. ملاحظات تقنية

- قراءة السيريال في Arduino أصبحت non-blocking لتقليل الرجفة على Linux/Radxa.
- كل أمر من الواجهة يرسل مع `\n` عبر دالة `sendCommand`.
- سلايدر `ACCELERATION` موجود بجانب `MOTOR SPEED` ويقسم نفس العمود بالتساوي.
- Arduino يتجاهل رسائل `STATUS`, `ERROR`, و `STEPS` إذا رجعت إليه بالخطأ من Node-RED.
- ريلاي الروبوت Active-Low: التفعيل بـ `LOW` والفصل بـ `HIGH`.
- بعد انتهاء كل دورة كاملة يرجع النظام إلى الهوم ثم يعود إلى `ARMED` لانتظار القطعة التالية.
